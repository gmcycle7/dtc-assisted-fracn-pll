Code Walkthrough

A line-by-line tour of the four model files behind this deck — how phase noise turns into jitter, how the type-II loop is solved, how the per-cycle time-domain PLL runs, and how a sign-LMS calibration converges — with every line mapped to a formula and a circuit node.

This page reads the actual Python in models/utils.py, models/frequency_domain_model.py, models/time_domain_noise_model.py, and models/calibration_models.py. Snippets are faithful to the source; long routines are trimmed to the essential lines with (...) marking omissions. Units are called out explicitly because almost every bug in PLL modeling is a units bug (rad vs cycles, single- vs double-sideband, Hz vs rad/s).

Orientation: how the four files fit together

The models share one vocabulary. Everything is SI: phase in rad, time in s, frequency in Hz slide p.42 sets the operating point ($f_\text{out}=6.72$ GHz, $f_\text{ref}=104$ MHz), and docs/assumptions.md sets the design knobs ($f_c=1.5$ MHz assumption [A1], PM $=60^\circ$ assumption [A2]).

utils.py

The shared toolbox: phase-noise ↔ jitter conversions, Welch PSD, noise-shape generators, DSM-QN PSD. No matplotlib import, so it runs headless.

frequency_domain_model.py

The small-signal s-domain model: solves the type-II loop, builds $G_\text{ol}$, $H_\text{ref}$, $H_\text{vco}$, integrates the jitter budget that reproduces slide p.42.

time_domain_noise_model.py

A discrete-time, per-reference-cycle behavioral PLL. Injects real random noise sequences, runs the loop, and Welch-PSDs the output to cross-check the frequency model.

calibration_models.py

The four background sign-LMS calibrations (DTC gain, VCO DCC, CKREF DCC, offset) plus the survey polynomial NLC.

1. utils.py — the phase-noise ↔ jitter chain

Every jitter number in this project flows through three functions. The chain is: $$ L(f)\ \xleftrightarrow{\;\texttt{Sphi\_to\_L}\;}\ S_\phi(f)\ \xrightarrow{\;\texttt{integrate\_phase\_noise}\;}\ \sigma_\phi\ \xrightarrow{\;\texttt{rms\_jitter\_from\_sigma\_phi}\;}\ \sigma_t. $$ Get any one of these wrong and the final fs number is wrong, so they are worth understanding exactly.

1a. Sphi_to_L — phase PSD to dBc/Hz

What it does: converts a single-sided phase power spectral density $S_\phi(f)$ (in $\text{rad}^2/\text{Hz}$) into the SSB phase-noise number $L(f)$ that everyone quotes in dBc/Hz. The factor of $\tfrac12$ is the SSB-from-DSB convention (IEEE-1139 / Leeson) textbook (IEEE-1139) assumption [A19].

def Sphi_to_L(Sphi):
Input $S_\phi$ in $\text{rad}^2/\text{Hz}$. This is the one-sided phase PSD at the output node after the loop has shaped it.
Sphi = np.asarray(Sphi, dtype=float)
Coerce to a float array so the function works on a scalar or a whole frequency sweep.
return 10.0 * np.log10(0.5 * np.clip(Sphi, 1e-300, None))
Formula: $L(f)=10\log_{10}\!\big(\tfrac12 S_\phi\big)$. The $0.5$ folds the double-sided variance into one sideband (the "carrier-to-single-sideband" ratio). The clip at $10^{-300}$ just keeps $\log_{10}(0)$ from blowing up. Circuit meaning: $L(f)$ is what a spectrum analyzer would read on one side of the 6.72 GHz carrier.
Its inverse is L_to_Sphi(L_dbc) = 2.0 * 10**(L_dbc/10) — note the factor of 2, undoing the $\tfrac12$. Every noise source in the frequency model is specified in dBc/Hz and immediately pushed through L_to_Sphi to get $S_\phi$.

1b. integrate_phase_noise — PSD to RMS phase

What it does: integrates the phase PSD over an offset band $[f_\text{lo},f_\text{hi}]$ to get the RMS phase $\sigma_\phi$ in radians. This is Parseval / the variance theorem textbook (Parseval): the area under $S_\phi$ is the phase variance.

freq = np.asarray(freq, dtype=float)
Sphi = np.asarray(Sphi, dtype=float)
Two parallel arrays: the offset-frequency grid (Hz, usually log-spaced) and the PSD on that grid.
m = np.ones_like(freq, dtype=bool)
if f_lo is not None: m &= freq >= f_lo
if f_hi is not None: m &= freq <= f_hi
Build a boolean mask selecting only the integration band. The default band for jitter is 1 kHz … 100 MHz assumption [A16]; the time-domain model narrows $f_\text{hi}$ to the 52 MHz Nyquist limit.
var = _trapz(Sphi[m], freq[m])
Formula: $\sigma_\phi^2=\int_{f_\text{lo}}^{f_\text{hi}} S_\phi(f)\,df$ by trapezoidal rule. _trapz is numpy's trapezoid (or legacy trapz) resolved at import — a numpy-2.0 compatibility shim. Trapezoid on a log grid is the standard way to integrate PSDs.
return float(np.sqrt(var))
Return $\sigma_\phi=\sqrt{\text{var}}$ in radians — the RMS phase wobble of the carrier over that band.

1c. rms_jitter_from_sigma_phi — RMS phase to RMS time

What it does: converts an RMS phase (rad) to an RMS timing jitter (s) at the carrier frequency. This is the one-liner that produces the headline 87.6 fs number.

def rms_jitter_from_sigma_phi(sigma_phi, f_out):
sigma_phi in rad, f_out in Hz (here 6.72 GHz).
return sigma_phi / (2.0 * np.pi * f_out)
Formula: $\sigma_t=\dfrac{\sigma_\phi}{2\pi f_\text{out}}$. One full carrier cycle is $2\pi$ rad and lasts $1/f_\text{out}$, so phase-to-time scales by $1/(2\pi f_\text{out})$. Numbers: $\sigma_\phi$ that integrates to the slide-42 budget gives $\sigma_t=87.6$ fs model ≈ 87.5 fs slide p42.
The full chain, end to end. Combine the three: a flat $L(f)=-160$ dBc/Hz over a 1 kHz…100 MHz decade-ish band first becomes $S_\phi = 2\cdot10^{-16}\,\text{rad}^2/\text{Hz}$, integrates to some $\sigma_\phi$, and divides by $2\pi(6.72\text{ GHz})$ to land in femtoseconds. The convenience wrapper integrated_jitter(freq, Sphi, f_out) does exactly this and returns (sigma_t, sigma_phi) in one call. The complementary ipn_dbc(sigma_phi, ssb=True) reports the same result as an integrated-phase-noise number, $\text{IPN}=10\log_{10}(\tfrac12\sigma_\phi^2)$, which lands at $-51.6$ dBc model ≈ $-51.7$ dBc slide p42.
The 3-dB SSB/DSB convention matters. ipn_dbc exposes both: ssb=True matches the deck's $-51.6$ dBc; ssb=False would report the full double-sideband variance and read 3 dB higher. The code comments flag this as a needs citation point in references.md — always confirm which convention a measured number uses before comparing.

2. frequency_domain_model.py — the type-II loop solve and the transfer functions

This file's job is to turn two design targets — loop bandwidth $f_c$ and phase margin PM — into a concrete open-loop gain $G_\text{ol}(s)$, then build the two closed-loop noise transfer functions. The cleverness is the symmetric zero/pole placement, which lets you solve the loop in closed form instead of iterating.

2a. _design_type2_loop — symmetric zero/pole solve

What it does: places one zero $\omega_z$ below crossover and one pole $\omega_p$ above crossover, geometrically symmetric about $\omega_c$, so the phase margin is maximized exactly at the chosen crossover. Then it scales the loop gain so $|G_\text{ol}(j\omega_c)|=1$ textbook (Gardner, symmetric type-II).

wc = 2 * np.pi * self.f_c
pm = np.deg2rad(self.pm_deg)
Crossover in rad/s ($\omega_c=2\pi\cdot1.5$ MHz) and the target phase margin in rad (60°).
k = np.tan((pm + np.pi / 2) / 2.0)
Formula: the symmetric type-II has $\text{PM}=2\arctan(k)-90^\circ$, so inverting gives $k=\tan\!\big(\tfrac{\text{PM}+90^\circ}{2}\big)$. The factor $k$ is the geometric "spread" of zero and pole around $\omega_c$. For PM $=60^\circ$, $k=\tan(75^\circ)\approx3.73$.
self.wz = wc / k
self.wp = wc * k
Placement: zero a factor $k$ below crossover, pole a factor $k$ above. They are geometrically symmetric: $\sqrt{\omega_z\omega_p}=\omega_c$. Circuit: $\omega_z$ comes from the proportional path (Vctrl_P, the loop-filter zero) slide p.39; $\omega_p$ is the extra parasitic/anti-spur pole.
jwc = 1j * wc
shape = (1 + jwc / self.wz) / (jwc ** 2 * (1 + jwc / self.wp))
Evaluate the unscaled loop shape at $s=j\omega_c$. The $s^2$ in the denominator is the two integrators: the VCO ($K_\text{vco}/s$) plus the loop's own integral term — the defining feature of a type-II loop.
self.Kloop = 1.0 / abs(shape)
Formula: choose $K_\text{loop}$ so $|G_\text{ol}(j\omega_c)|=1$, i.e. $K_\text{loop}=1/|\text{shape}(j\omega_c)|$. This forces the crossover to land exactly at $f_c$. The numeric sanity check loop_metrics later recovers $f_c=1.501$ MHz, PM $=60^\circ$ slide p.42 / json, confirming the closed-form solve.

2b. open_loop, H_ref, H_vco — the transfer functions

What they do: assemble $G_\text{ol}(s)$ from the solved constants, then form the two closed-loop shapes. H_ref is low-pass (the reference and everything injected with it pass through the loop at DC, gained up by $N$); H_vco is high-pass (the loop suppresses VCO noise inside the bandwidth and lets it through outside).

s = 1j * 2 * np.pi * np.asarray(f, dtype=float)
return params.Kloop * (1 + s / params.wz) / (s ** 2 * (1 + s / params.wp))
open_loop: $G_\text{ol}(s)=K_\text{loop}\dfrac{1+s/\omega_z}{s^2\,(1+s/\omega_p)}$. The numerator zero supplies the phase boost for stability; the double pole at the origin (the $s^2$) gives the type-II's zero steady-state phase error. Circuit: one $1/s$ is the VCO integrator, the other is the loop-filter integral path (Vctrl_I) slide p.39.
G = open_loop(params, f)
return params.N * G / (1 + G)
H_ref $=N\dfrac{G}{1+G}$. Classic feedback: at low offset $G\!\gg\!1$ so $H_\text{ref}\to N$ (DC gain $=N=64.62$, because the reference is multiplied up to the output). At high offset $G\to0$ so it rolls off — a low-pass shape. REF, DTC, MMD, and SPD+GM noise all ride this path.
G = open_loop(params, f)
return 1.0 / (1 + G)
H_vco $=\dfrac{1}{1+G}$. At low offset $G\!\gg\!1$ so $H_\text{vco}\to0$ (loop crushes VCO noise in-band); at high offset $G\to0$ so $H_\text{vco}\to1$ (free-running VCO leaks straight through) — a high-pass shape. This is why the VCO dominates the far-out budget at 51.8% slide p.42.
The combined budget that these transfer functions produce — output_contributions sums each source's $S_\phi$ times $|H|^2$, and jitter_budget integrates them — reproduces the deck pie: VCO 51.8%, REF+DTC ≈37.8%, MMD 6.2%, SPD+GM 3.9%, DSM-QN ≈0.3% (cancelled), totaling 87.6 fs model ≈ 87.5 fs slide p42 / $-51.6$ dBc model ≈ $-51.7$ dBc slide p42.

3. time_domain_noise_model.py — the per-cycle loop (the heart)

This is the most important routine in the project: a genuine discrete-time simulation of the PLL, sampled once per reference cycle ($f_s=f_\text{ref}=104$ MHz, Nyquist $=52$ MHz). Instead of multiplying PSDs by transfer functions, it injects actual random noise sequences at the right nodes and lets the loop evolve. If the resulting Welch-PSD matches the frequency-domain model, both are trustworthy — and they do (88.2 fs vs 87.4 fs, $+0.08$ dB) derived.

Before the loop, each noise source is generated as a sequence referred to its injection node (rad): reference (white floor + flicker), DTC (QN + thermal, white), SPD+GM (white), MMD (white), and the free-running VCO. The VCO is special — it enters as the per-cycle increment:

vco_step = np.diff(vco_noise, prepend=vco_noise[0])
The VCO accumulates phase, so its noise appears as the step added each cycle. np.diff with prepend keeps the length at $n$. Circuit: the VCO is a phase integrator; injecting the increment vco_step[k] and then accumulating reproduces the $-20$ dB/dec white-FM shape.

3a. The five-step per-cycle loop body

What it does: walks one reference cycle. Each numbered step is a physical block: VCO accumulator → feedback divider → reference+DTC path → sampling phase detector → PI loop filter. The loop runs for $n=2^{20}\approx1.05$M cycles.

for k in range(1, n):
Iterate cycle by cycle from $k=1$ (state at $k=0$ is the initial condition). Index $k$ is one reference period $T_s=1/f_\text{ref}\approx9.6$ ns.
phi_out[k] = phi_out[k-1] + Kv * ctrl + vco_step[k]
Step 1 — VCO phase accumulation. $\varphi_\text{out}[k]=\varphi_\text{out}[k-1]+K_v\,\text{ctrl}+\Delta\varphi_\text{vco}[k]$. The previous control voltage integrates into output phase ($K_v$ absorbed to 1), and the VCO's own noise increment is added. This is the discrete VCO integrator $K_v z^{-1}/(1-z^{-1})$.
phi_fb = phi_out[k] / N + mmd_noise[k]
Step 2 — feedback divider (/N). $\varphi_\text{fb}=\varphi_\text{out}/N+\Delta\varphi_\text{mmd}$. The MMD divides the output phase by $N=64.62$ and adds its own input-referred noise slide p.42. Circuit: the multi-modulus divider in the feedback path.
phi_ref = ref_noise[k] + dtc_noise[k]
Step 3 — reference + DTC path. $\varphi_\text{ref}=\Delta\varphi_\text{ref}+\Delta\varphi_\text{dtc}$. Critically the DSM quantization error is not added — the DTC cancels it slide p.9 — only the DTC's residual QN + thermal noise remains.
e = phi_ref - phi_fb + spd_noise[k]
Step 4 — sampling phase detector. $e=\varphi_\text{ref}-\varphi_\text{fb}+\Delta\varphi_\text{spd}$. The SPD measures the phase error between reference and feedback and adds its own + the GM front-end noise slide p.20. This $e$ is the loop's error signal in rad.
acc += e
ctrl = Kp * e + Ki * acc
Step 5 — PI loop filter (type-II). The integral accumulator builds up $\sum e$; the control is $\text{ctrl}=K_p e+K_i\sum e$. The proportional term $K_p$ is the stabilizing zero, the integral term $K_i$ gives the second pole at origin. Circuit: Vctrl_P + Vctrl_I summed into the VCO slide p.39. The next iteration's step 1 feeds ctrl back into the VCO — closing the loop.
Where $K_p$ and $K_i$ come from. The discrete loop (in DTLoop.__post_init__) is matched to the same $f_c$/PM as the frequency model: $\omega_z=\omega_c/\tan(\text{PM})$, $K_c=\omega_c/\sqrt{1+(\omega_z/\omega_c)^2}$ to force $|L(j\omega_c)|=1$, then mapped to PI gains $K_p=K_c N T_s/K_v$ and $K_i=K_p\,\omega_z T_s$. Because both models target identical $f_c$ and PM, their outputs must agree — which is the cross-check derived.
The time-domain Nyquist is only 52 MHz (it samples at $f_\text{ref}$), so its jitter is integrated over 1 kHz…52 MHz. The frequency model's 100-MHz band picks up a little extra VCO tail, which is why the two numbers differ by a hair (88.2 vs 87.4 fs). This band limitation is documented in docs/open_questions.md.

4. calibration_models.py — sign-LMS in one loop body

All four background calibrations share one update law derived: $$ w[k+1]=w[k]+\mu\,e[k]\,x[k],\qquad e[k]=\operatorname{sign}(\text{PHE}[k]), $$ where $e[k]$ is a 1-bit comparator output and $x[k]$ is the regressor. The 1-bit error is what makes these cheap to implement on-chip (no full ADC) but slower to converge. We walk the DTC-gain version because it is the one that, if left uncalibrated, leaks DSM quantization energy back into the band.

4a. simulate_dtc_gain_cal loop body — residual → comparator → LMS update

What it does: the DTC cancels the accumulated DSM quantization error by injecting a delay proportional to $\hat K_\text{DTC}\cdot\Phi_e$. If the gain estimate $\hat K$ is wrong, a residual phase error proportional to $\Phi_e$ remains; the sign-LMS loop correlates the sign of that residual with $\Phi_e$ to push $\hat K$ toward the true gain.

Khat = K_true * (1.0 - K_init_err)
Start the estimate 10% low assumption [A18]: $\hat K=K_\text{true}(1-0.10)=900$ for $K_\text{true}=1000$ codes/cycle (the DTC gain $=T_\text{vco}/\text{LSB}$).
for k in range(n_cycles):
One iteration per reference cycle — the cal update rate is $f_\text{ref}$. cycles_for_time converts µs targets to cycle counts.
phi_e = rng.uniform(0.0, 0.5) if half_range else rng.uniform(-0.5, 0.5)
Regressor. $\Phi_e$ is the accumulated DSM quantization error this cycle, in cycles. Half-range DSM (slide 34) keeps it in $[0,0.5]$ so the sign of the residual is unambiguously tied to the gain error.
phe = (K_true - Khat) / K_true * phi_e
Residual. $\text{PHE}=\dfrac{K_\text{true}-\hat K}{K_\text{true}}\,\Phi_e$. The normalized gain mismatch scales $\Phi_e$ into a leftover phase error. When $\hat K\to K_\text{true}$, PHE $\to0$ — the cal is done. Circuit: this is the SPD output after imperfect DTC cancellation slides p.26-28.
if comp_noise_lsb:
    phe += rng.normal(0.0, comp_noise_lsb / K_true)
Optional comparator input noise, expressed in LSB of the residual and divided by $K_\text{true}$ to normalize into the same PHE units. It dithers the sign decision and sets the converged limit-cycle floor.
e = np.sign(phe + offset / K_true) or 1.0
1-bit comparator. $e=\operatorname{sign}(\text{PHE}+\text{offset})$ slide p.27. The or 1.0 guards the $\operatorname{sign}(0)=0$ case. A non-zero offset biases the sign and drives $\hat K$ to the wrong value — exactly why the separate offset cal (4 below) exists.
Khat += mu * e * phi_e
Sign-error LMS update. $\hat K\!\leftarrow\!\hat K+\mu\,e\,\Phi_e$ — the shared law with $w=\hat K$, regressor $x=\Phi_e$. Step size $\mu=0.5$ assumption [A17]. Because $e$ correlates with the sign of the gain error, the update statistically moves $\hat K$ uphill toward $K_\text{true}$.
Convergence (all < 30 µs) slide p.40: DTC gain 6.7 µs, VCO DCC 1 µs (target 10 ps for a 20 ps duty error), CKREF DCC 1 µs (target 0.337 ns), offset 14.6 µs (target 32 mV). The offset cal converts its error to phase via $K_\text{SPD}=K_\text{slope}/(2\pi f_\text{ref})=9.19$ V/rad slide p.20.
The sibling cals (simulate_vco_dcc, simulate_ckref_dcc) reuse the same body with a deterministic ±1 regressor instead of $\Phi_e$: VCO DCC correlates $e[k]$ with SEL_CKFB (toggles each cycle) to estimate $\Delta t_\text{err}/2$; CKREF DCC correlates with even_cycle. The offset cal is a DC servo on the leaky mean of $e[k]$ rather than a regressor product. These are behavioral assumptions from docs/assumptions.md chosen to reproduce the slide-40 settling, not measured silicon traces needs citation.

Takeaways

One conversion chain

$L \leftrightarrow S_\phi \to \sigma_\phi \to \sigma_t$, with the $\tfrac12$ SSB factor and the $1/(2\pi f_\text{out})$ scaling. Everything else feeds it.

Closed-form loop

Symmetric zero/pole ($\omega_z=\omega_c/k$, $\omega_p=\omega_c k$, $k=\tan\tfrac{\text{PM}+90^\circ}{2}$) means no iteration — the solve lands $f_c$ and PM exactly.

Five-step heart

VCO accumulate → /N feedback → ref+DTC → SPD error → PI filter. Inject real noise, Welch-PSD the output, cross-check the frequency model.

One update law

$w[k+1]=w[k]+\mu\,e[k]\,x[k]$ with a 1-bit $e[k]$ powers all four cals; only the regressor changes.
